# GIT & Github

- `git add .` - Добавляет все изменения из рабочей директории в область подготовки (staging area).

- `git reset` - Отменяет изменения в области подготовки, но оставляет их в рабочей директории.

- `git reset --hard` - Отменяет и удаляет все изменения, как в области подготовки, так и в рабочей директории, возвращаяся к состоянию последнего коммита. Будьте осторожны, это удалит несохраненные изменения.

**Команда `git restore` в Git**

Команда `git restore` используется в Git для восстановления файлов в рабочей директории до предыдущего состояния. Она предоставляет возможность отменить изменения в файлах, восстановив их до состояния последнего коммита или до состояния в области подготовки (staging area).

**Как использовать `git restore`:**

1. *Восстановление файла до состояния в последнем коммите:*

   - `git restore имя-файла` отменяет все изменения в файле и восстанавливает его до состояния в последнем коммите. Это может быть полезно, если вы внесли изменения в файл и хотите отменить их.

   - `git restore --staged имя-файла` отменяет изменения в файле и восстанавливает его до состояния в области подготовки. Это позволяет отменить добавление файла в область подготовки перед коммитом.

    - Вы можете использовать маски файлов, например, `git restore "*.txt"`, чтобы восстановить несколько файлов сразу.

    - `git restore --source коммит имя-файла` позволяет вам восстановить файл до состояния, указанного в конкретном коммите. Это полезно, если вы хотите восстановить файл до определенной версии.

    - `git restore --worktree имя-файла` может быть использована, чтобы явно указать, что вы хотите восстановить файл до состояния в рабочей директории.

Команда git restore является частью Git версии 2.23 и новее. Если вы используете более раннюю версию Git, вы можете использовать команду git checkout с аналогичными опциями для восстановления файлов.
