# GIT & Github Handbook

### `git add`

Команда `git add` используется для добавления изменений в рабочей директории в индекс (или стэйдж), готовя их к коммиту.

#### Описание
Команда `git add` позволяет выбирать, какие изменения в рабочей директории вы хотите добавить в следующий коммит.

#### Использование
- `git add <file>`: Добавляет конкретный файл в индекс.
- `git add .`: Добавляет все измененные файлы в индекс.
- `git add --all`: Добавляет все измененные файлы и удаленные файлы в индекс.
- `git add -p`: Интерактивно добавляет изменения по частям.

#### Пример

`git add index.html` Добавляет изменения в файле index.html в индекс

`git add .`  Добавляет все изменения в текущем каталоге в индекс

`git add -p`  Позволяет выбирать изменения для добавления интерактивно


После использования `git add`, изменения будут готовы для коммита при выполнении `git commit`.

### `git reset`

Команда `git reset` используется для изменения состояния коммитов в репозитории.

#### Описание

`git reset` позволяет изменять состояние индекса и рабочего каталога, отменяя коммиты, изменяя коммиты или отменяя добавление файлов в индекс.

#### Использование

- `git reset <commit>`: Сбрасывает HEAD до указанного коммита, удаляя коммиты после него.
- `git reset --soft <commit>`: Сбрасывает HEAD до указанного коммита, оставляя изменения в индексе.
- `git reset --mixed <commit>`: Сбрасывает HEAD до указанного коммита, удаляя изменения из индекса.
- `git reset --hard <commit>`: Сбрасывает HEAD до указанного коммита, удаляя изменения из индекса и рабочего каталога.

#### Пример

```bash
git reset HEAD~1              # Сбрасывает HEAD на предыдущий коммит, оставляя изменения в рабочем каталоге
git reset --hard HEAD~2       # Удаляет последние два коммита и любые изменения в них из индекса и рабочего каталога
```

### Предостережение
Будьте осторожны с использованием git reset --hard, так как это безвозвратно удаляет изменения в рабочем каталоге.

### `git restore`

Команда `git restore` используется для отмены изменений в рабочем каталоге.

#### Описание

`git restore` позволяет отменить изменения, внесенные в рабочий каталог, возвращая его к состоянию последнего коммита.

#### Использование

- `git restore <file>`: Отменяет изменения в указанном файле, восстанавливая его до последнего коммита.
- `git restore .`: Отменяет все изменения в рабочем каталоге, возвращая все файлы к состоянию последнего коммита.

#### Пример

```bash
git restore index.html              # Отменяет изменения в файле index.html
git restore .                       # Отменяет все изменения в рабочем каталоге
